# Kwork-бот — быстрый старт

Прототип: `PROTOTYPE_BOT.py`. Анализ: `KWORK_API_ANALYSIS.md`. Архитектура: `IMPLEMENTATION_PLAN.md`.

> ⚠️ Автоответы на бирже могут нарушать ToS Kwork (риск блокировки аккаунта). Каждый отклик списывает 1 коннект. Используйте задержки и лимиты.

## 1. Установка

```bash
pip install requests playwright
playwright install chromium
```

## 2. Первый вход (один раз) — сохранить сессию kwork.ru

```bash
python PROTOTYPE_BOT.py login
```

Откроется браузер на `kwork.ru/login`. Войдите **вручную** (логин/пароль, капча, SMS/2FA),
дождитесь загрузки кабинета и нажмите Enter в терминале. Создастся `storage_state.json` —
файл с cookie сессии, который дальше переиспользуется.

## 3. Настройка

В `PROTOTYPE_BOT.py`, блок `__main__` → `Config(...)`:

| параметр | смысл |
|---|---|
| `login` / `password` | креды для REST-чтения списка заявок (`/signIn`) |
| `price_from` / `price_to` | бюджет заявок для выборки |
| `query` / `categories` | ключевые слова / id рубрик (фильтр) |
| `offer_price` | сумма отклика (1200–18000) |
| `offer_duration_days` | срок в днях — из `[1,2,3,4,5,6,7,10,14,21,30,60]` |
| `offer_name` | заголовок оффера |
| `max_offers_per_run` | лимит откликов за прогон |
| `min_delay_s` / `max_delay_s` | случайная пауза между откликами |
| `dry_run` | `True` = заполнить форму, но НЕ отправлять (тест); `False` = реально отправлять |

Также отредактируйте `build_offer_text()` — уникализируйте текст под каждую заявку.

## 4. Прогон

```bash
# тест: заполняет форму, НИЧЕГО не отправляет (dry_run=True по умолчанию)
python PROTOTYPE_BOT.py run

# боевой режим: поставьте dry_run=False в Config, затем
python PROTOTYPE_BOT.py run
```

`run` (Playwright, рекомендуется): REST читает заявки → фильтр/дедуп → форма заполняется
и отправляется в браузере с сохранённой сессией.

`run-http` — альтернатива через `POST /api/offer/createoffer` (быстрее, но хрупче;
требует cookie-сессии kwork.ru в requests и работающего `csrftoken` со страницы).

## 5. Что обязательно проверить перед боем

1. `dry_run=True` — убедитесь, что форма заполняется правильно (текст/сумма/срок видны в браузере).
2. Один контролируемый сабмит (`dry_run=False`, `max_offers_per_run=1`) — проверьте ответ
   `createoffer` и что коннект списался корректно.
3. Селекторы могут поменяться при редизайне сайта — тогда обновите их в `submit_offer_playwright()`
   (см. список селекторов в шапке файла и в `KWORK_API_ANALYSIS.md §8`).

## Файлы, создаваемые ботом

- `storage_state.json` — cookie сессии (после `login`).
- `uad.txt` — стабильный device id для REST.
- `state.json` — id уже отвеченных заявок (антиповтор).
