# Kwork Auto-Responder Bot

Автономный бот для биржи фриланса [Kwork](https://kwork.ru): каждый час читает реальный
остаток коннектов, находит **новые** заявки в нужной категории, через LLM
(Groq · `llama-3.3-70b`) пишет персональное профессиональное предложение,
сам проставляет цену и срок и отправляет отклик через залогиненный браузер.

> ⚠️ Проект исследовательский. Автоматизация откликов может нарушать правила
> Kwork — используйте на свой риск. Никаких секретов в репозитории нет, все
> ключи и пароли задаются через `.env` (см. `.env.example`).

## Возможности

- **Балансо-ориентированный запуск.** Каждый час бот читает «Осталось N из 30»
  прямо с биржи. `N = 0` → спит до следующего часа; `N > 0` → отвечает не больше,
  чем на N новых заявок.
- **Только новые заявки.** Состояние (`seen` / `answered`) хранится в
  `connects_state.json`, повторно одни и те же заявки не дёргаются.
- **Профильный фильтр в два слоя:**
  1. жёсткий стоп-лист по ключевым словам (Tilda, Wix, WordPress, Bitrix и т.п.);
  2. решение LLM — берётся только самописная разработка (чистый код), мусорные и
     непрофильные заявки пропускаются.
- **Профессиональный текст.** LLM не пересказывает заявку и не обращается по нику —
  предлагает конкретное техническое решение, стек и план, 350–650 символов.
- **Системное ценообразование.** Цена всегда = `макс. бюджет − 10%`
  (округление до 10 ₽, не ниже пола `KWORK_PRICE_FLOOR`).
- **Headless-браузер + CDP.** Отправка идёт через Playwright, подключённый к
  запущенному Chrome (CSRF, авто-портфолио и Vue-форма обрабатываются как у человека).
- **Запуск по расписанию.** `cron` каждый час + `flock` против наложения прогонов.

## Архитектура

```
exchange (kwork.ru/projects?c=11)        Groq LLM (llama-3.3-70b)
        │ список новых заявок                    │ текст / срок / название
        ▼                                        ▼
 batch_autopilot.py  ──►  ai_autopilot.py  ──►  Playwright (CDP)  ──►  kwork.ru
   (оркестратор:           (фильтр + LLM +        (заполнение и
    баланс, пул,            цена −10%)             отправка формы)
    состояние)
```

## Файлы

| Файл | Назначение |
|------|------------|
| `batch_autopilot.py` | Почасовой оркестратор: читает баланс, собирает новые заявки, ведёт состояние, обходит пул |
| `ai_autopilot.py` | Открывает заявку, фильтрует, зовёт LLM, считает цену, заполняет и отправляет форму |
| `server_login.py` | Первичный логин в Kwork в headless-Chrome и сохранение сессии |
| `serve_chrome.sh` | Запуск Chrome с CDP-эндпоинтом (для systemd-службы) |
| `groq_proxy_worker.js` | Cloudflare Worker — прокси к Groq API, если прямой доступ заблокирован |
| `scrape_list.py` | Вспомогательный скрипт сбора id заявок с биржи |
| `PROTOTYPE_BOT.py` | Ранний прототип (REST + Playwright, dry-run) |
| `KWORK_API_ANALYSIS.md` | Анализ REST/веб-API Kwork и реального эндпоинта сабмита |
| `IMPLEMENTATION_PLAN.md` | Архитектура, эндпоинты, анти-бан |
| `QUICKSTART.md` | Быстрый старт |

## Установка

```bash
git clone https://github.com/rinat3636/kwork-auto-responder.git
cd kwork-auto-responder
python -m venv venv && source venv/bin/activate
pip install requests playwright
playwright install chromium

cp .env.example .env   # заполните своими значениями
```

## Запуск

```bash
# 1. поднять Chrome с CDP
bash serve_chrome.sh

# 2. первичный логин (сохранит сессию в профиль браузера)
python server_login.py

# 3. один прогон вручную (dry-run для теста):
python batch_autopilot.py 5      # ограничить число откликов

# 4. боевой запуск по расписанию (cron, каждый час):
#    0 * * * * /path/to/run_hourly.sh
```

## Конфигурация (`.env`)

| Переменная | По умолчанию | Описание |
|-----------|--------------|----------|
| `KWORK_LOGIN` / `KWORK_PASSWORD` | — | Логин в Kwork (только для первичного входа) |
| `GROQ_API_KEY` | — | Ключ Groq |
| `GROQ_MODEL` | `llama-3.3-70b-versatile` | Модель LLM |
| `GROQ_URL` | api.groq.com | Эндпоинт (можно указать прокси) |
| `CDP_URL` | `http://localhost:29229` | CDP запущенного Chrome |
| `KWORK_PRICE_DISCOUNT` | `0.10` | Скидка от макс. бюджета |
| `KWORK_PRICE_FLOOR` | `500` | Минимальная цена оффера |
| `SCAN_CAP` | `80` | Сколько новых заявок сканировать за прогон |
